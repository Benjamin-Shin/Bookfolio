# TODO 일지 (일자별)

## 목적

변경 의도와 범위를 남기고, 검토·승인 후 구현하는 흐름을 맞추기 위한 기록입니다.

## 사용 방법

1. 작업할 날짜로 파일을 둡니다: `YYYY-MM-DD.md`
2. 항목을 적고 상태를 표시합니다: `draft` → (검토) → `approved` → 구현 후 `done`
3. 구현이 끝나면 코드 쪽에 JSDoc `@history`를 남깁니다 (Cursor 규칙 `workflow-todo-approval.mdc` 참고)

## 템플릿

새 날짜 파일을 만들 때 `_template.md`를 복사해 이름만 바꿔 사용할 수 있습니다.
